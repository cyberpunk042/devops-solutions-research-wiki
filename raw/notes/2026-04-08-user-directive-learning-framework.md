## Interpretation

### 1. Learning Framework
Ingestion is not terminal — it's the start. Each ingested source should spawn:
- Structured lessons (what was learned, per domain)
- Evolution of ideas (how concepts connect, contradict, merge)
- Aggregations (synthesized views across multiple sources)
- Filterings and orderings (ranked by confidence, relevance, maturity)
- Annotations and enrichments (metadata, cross-references, quality scores)

### 2. Knowledge Evolution Pipeline
Sources → synthesis → lessons → patterns → principles → decisions. Each layer builds on the previous:
- Layer 0: Raw sources (raw/)
- Layer 1: Source syntheses (wiki/sources/)
- Layer 2: Concept pages (wiki/domains/)
- Layer 3: Comparisons + cross-references (wiki/comparisons/)
- Layer 4: Lessons learned (new — structured insights per domain)
- Layer 5: Patterns + principles (new — aggregated wisdom)
- Layer 6: Decision frameworks (new — actionable guidance)

### 3. Strategic Templates
Predefined page templates that ensure consistency and depth:
- Lesson template (what, why, evidence, domain, confidence)
- Pattern template (pattern name, instances, when to apply, when not to)
- Decision template (decision, alternatives, rationale, reversibility)
- Domain overview template (state of knowledge, gaps, priorities)
- Evolution template (how a concept has changed over time)

### 4. Automation + Deterministic + $0
- Move toward local LLM inference for mechanical operations
- Deterministic pipelines for repeatable operations (no LLM randomness)
- $0 cost target for routine operations (local models, no API calls)
- LLM only for reasoning, connections, questions — everything else automated

### 5. Clean Wiki / Clean App
- Consistent structure everywhere
- Templates enforced by tooling
- Quality gates that prevent low-quality pages
- Spine of the vault = strategic pages that define the knowledge architecture
