## Key Directives to Remember

All logged verbatim in wiki/log/. The critical ones:
- **Follow the workflow** — document → design → scaffold → implement → test. Actually DO it, don't skip.
- **Never fabricate** — if you don't have measured data, say "unverified" or "one practitioner observed"
- **Depth verification** — read the THING, not the description of the thing (Layer 0 → Layer 1)
- **Models are not reading lists** — they define SYSTEMS with standards, schemas, examples
- **Three standard layers** — structure (schema) + quality (standards page) + visual design (callout vocabulary)
- **Three formatting contexts** — universal markdown (baseline), Obsidian (wiki), remark/Docusaurus (public docs). Different contexts, not tiers.
- **The wiki is the HUB** — aggregates learnings from all projects, feeds knowledge back
